# Types & Syntax

EpochScript type system, variable assignment, and parameter syntax fundamentals.

---

## Supported Types

`Boolean`, `Integer`, `Decimal`, `String`, `None`

**Type aliases:** `Number` (Integer or Decimal), `Timestamp` (from data source), `Any` (generic)

<Callout type="warning">
**Strings are isolated** - can only be used for labels and conditional selection, not arithmetic operations.
</Callout>

---

## Variables

**Assignment:** Single assignment only - variables are immutable.

**Naming rules:**
- Must start with letter or underscore: `ema_20`, `_temp`, `Signal`
- Can contain letters, numbers, underscores: `fast_ema_12`, `rsi_14`
- Case-sensitive: `Price` ≠ `price`
- Cannot use Python reserved keywords: `True`, `False`, `None`, `and`, `or`, `not`, `if`, `else`

**Reassignment is prohibited:**
```python
price = src.c
price = src.o  # ERROR: Variable 'price' already bound
```

---

## Transform Parameters (Options)

**Parameter typing is strict for String and Boolean types.** Numeric types (Integer, Decimal, Number) are flexible and interchangeable.

| Type       | Valid                      | Invalid (causes error)        |
|------------|----------------------------|-------------------------------|
| `Integer`  | `20`, `-5`, `20.0`         | `"20"`, `True`                |
| `Decimal`  | `3.14`, `0.5`, `20`        | `"3.14"`, `False`             |
| `Number`   | `20`, `3.14`, `-5.0`       | `"42"`, `True`                |
| `Boolean`  | `True`, `False`            | `1`, `0`, `"true"`            |
| `String`   | `"1D"`, `"AAPL"`           | `1D` (missing quotes)         |

<Callout type="note">
**Automatic type compatibility:**
- **Numeric types are interchangeable**: Integer ↔ Decimal ↔ Number
- **Strings must be quoted**: Bare identifiers cause errors
- **Booleans are strict**: Use `True`/`False`, not `1`/`0`
</Callout>

---

## Type Casting

EpochScript automatically casts between compatible types in expressions:

**Numeric Compatibility:**
- `Integer`, `Decimal`, and `Number` are mutually compatible
- Operations mix types freely: `20 + 3.14` works without explicit casting

**Boolean ↔ Numeric Casting:**
- Boolean → Numeric: `True` becomes `1.0`, `False` becomes `0.0`
- Numeric → Boolean: Non-zero values become `True`, zero becomes `False`

**String Isolation:**
- Strings cannot be cast to other types
- Use only for labels, conditional selection, and metadata

---

## Multi-Output Transforms

Transforms with multiple outputs (like `bbands`, `macd`, `stoch`) require explicit output selection.

| Method | Example |
|--------|---------|
| **Attribute access** | `bb.bbands_upper` |
| **Tuple unpacking** | `lower, middle, upper = bbands(...)` |
| **Discard unwanted** | `lower, _, upper = bbands(...)` |

**Rules:**
- Unpacking count must match output count exactly (or use `_`)
- Can only unpack direct transform calls, not variables
- Error: `ambiguous_multi_output` means you need to specify which output
