# Limitations

EpochScript looks like Python but has strict constraints for vectorized time-series processing. Understanding these limitations is critical.

---

## Not Python

EpochScript is **declarative**, not imperative. You describe *what* you want, not *how* to compute it.

<Callout type="info">
EpochScript is NOT Python. Many Python features are intentionally excluded to enable vectorized execution across time series.
</Callout>

---

## No Control Flow

`if`/`elif`/`else` statements are not allowed.

**Reason:** Operations must work vectorized across all time points simultaneously.

<Callout type="warning" title="If statements not supported">
Use boolean expressions or ternary operators instead.
</Callout>

**Use instead:**

<!-- snippet: type=complete_code, needs_data=single_asset, complexity=simple -->
```python
src = market_data_source(timeframe="1D")
price = src.c
rsi_val = rsi(period=14)(src.c)

# Boolean expression
signal = price > 100

# Ternary operator
signal_ternary = True if price > 100 else False

# conditional_select for multi-way branching
regime = conditional_select(
    rsi_val < 30, "oversold",
    rsi_val > 70, "overbought",
    "neutral"
)
```

---

## No Loops

`for` and `while` loops are not allowed.

**Reason:** Transforms operate on entire series vectorized.

<Callout type="warning" title="Loops not supported">
Use vectorized transforms instead.
</Callout>

**Use instead:**

<!-- snippet: type=complete_code, needs_data=multi_asset, complexity=complex -->
```python
src = market_data_source(timeframe="1D")
close = src.c
momentum = rsi(period=14)(close)

# Transforms work on entire series automatically
ema_series = ema(period=20)(close)  # Applies to all time points

# Cross-sectional operations
top_stocks = top_k(k=10)(momentum)  # Selects top 10 across universe
```

---

## No Variable Reassignment

Variables are **immutable** - single assignment only.

**Reason:** Prevents temporal logic bugs and enables parallel evaluation.

<Callout type="warning" title="Variable already bound">
Use distinct variable names.
</Callout>

**Valid Example:**

<!-- snippet: type=complete_code, needs_data=single_asset, complexity=simple -->
```python
src = market_data_source(timeframe="1D")

# Valid - distinct names
close_price = src.c
open_price = src.o
```

**Invalid Example:**

<!-- snippet: type=error_example, needs_data=single_asset, complexity=simple -->
```python
src = market_data_source(timeframe="1D")

# NOT ALLOWED
price = src.c
price = src.o  # ERROR: Variable 'price' already bound
```

---

## No Function Definitions

`def` keyword is not allowed. Cannot define custom functions.

**Reason:** All transforms must be registered in the system for metadata and visualization.

<Callout type="note">
Use built-in transforms only. If you need custom logic, request a new transform to be added to the system.
</Callout>

---

## No Class Definitions

`class` keyword is not allowed.

**Reason:** EpochScript is a pure functional language for time-series operations.

---

## No Exception Handling

`try`/`except` blocks are not allowed.

**Reason:** All errors are compile-time errors, not runtime exceptions.

---

## No Chained Comparisons

Python's `a < b < c` syntax is not supported.

<Callout type="warning" title="Chained comparison not supported">
Use explicit boolean logic.
</Callout>

**Invalid Example:**

<!-- snippet: type=error_example, needs_data=single_asset, complexity=simple -->
```python
src = market_data_source(timeframe="1D")
rsi_val = rsi(period=14)(src.c)

# NOT ALLOWED
valid = 30 < rsi_val < 70
```

**Valid Example:**

<!-- snippet: type=complete_code, needs_data=single_asset, complexity=simple -->
```python
src = market_data_source(timeframe="1D")
rsi_val = rsi(period=14)(src.c)

# Use instead
above_30 = rsi_val > 30
below_70 = rsi_val < 70
valid = above_30 and below_70

# Or inline
valid_inline = (rsi_val > 30) and (rsi_val < 70)
```

---

## No Negative Indexing

Python's `list[-1]` for last element is not supported.

<Callout type="warning" title="Negative lag indices not supported">
Lag operator only accepts positive integers.
</Callout>

**Invalid Example:**

<!-- snippet: type=error_example, needs_data=single_asset, complexity=simple -->
```python
src = market_data_source(timeframe="1D")

# NOT ALLOWED
yesterday = src.c[-1]
```

**Valid Example:**

<!-- snippet: type=complete_code, needs_data=single_asset, complexity=simple -->
```python
src = market_data_source(timeframe="1D")

# Use instead
yesterday = src.c[1]  # 1 bar ago
last_week = src.c[5]  # 5 bars ago
```

---

## No Zero Indexing

Using `[0]` is redundant and not allowed.

<Callout type="warning" title="Zero lag not allowed">
Omit the index for current value.
</Callout>

**Invalid Example:**

<!-- snippet: type=error_example, needs_data=single_asset, complexity=simple -->
```python
src = market_data_source(timeframe="1D")

# NOT ALLOWED
current = src.c[0]
```

**Valid Example:**

<!-- snippet: type=complete_code, needs_data=single_asset, complexity=simple -->
```python
src = market_data_source(timeframe="1D")

# Use instead
current = src.c
```

## Summary Table

| Python Feature | Supported? | Alternative |
|----------------|------------|-------------|
| `if`/`else` | No | Ternary, `conditional_select()` |
| `for`/`while` | No | Vectorized transforms |
| Variable reassignment | No | Use distinct names |
| `def` functions | No | Use built-in transforms |
| `class` | No | N/A |
| `try`/`except` | No | Compile-time error checking |
| Chained comparisons | No | Explicit `and`/`or` |
| Negative indexing | No | Positive lag only `[N]` |
| Zero indexing | No | Omit index |

---

## Why These Limitations?

These constraints enable:

1. **Vectorized execution** - Operations across entire time series
2. **Compile-time safety** - Catch errors before execution
3. **Automatic visualization** - System knows about all operations
4. **Parallel evaluation** - No temporal dependencies from mutation
5. **Type safety** - Strong type checking without runtime overhead

The limitations aren't arbitrary - they're essential for EpochScript's design goals of safe, fast, visualizable time-series operations.
