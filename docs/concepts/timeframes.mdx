# Timeframes

Multi-timeframe analysis through automatic data resampling. Execute transforms on any timeframe regardless of strategy base resolution.

---

## Syntax

{/* snippet: type=complete_code, needs_data=single_asset, timeframe=TF, complexity=simple */}
```python
transform(period=N, timeframe="TF")(input)
```

The framework automatically resamples data to the specified timeframe before transform execution.

**Example:**
{/* snippet: type=complete_code, needs_data=single_asset, timeframe=1D, needs_intraday=true, complexity=medium */}
```python
# Strategy runs on 15Min bars
src = market_data_source(timeframe="1D")

# Calculate on daily timeframe (auto-resampled)
daily_ema = ema(period=50, timeframe="1D")(src.c)

# Compare intraday price vs daily trend
above_trend = src.c > daily_ema
```

## Standard Timeframes

### Intraday
{/* snippet: type=complete_code, needs_data=single_asset, needs_intraday=true, complexity=medium */}
```python
"1Min", "2Min", "3Min"     # 1-3 minutes
"5Min", "10Min"            # 5-10 minutes
"15Min", "30Min", "45Min"  # 15-45 minutes
"1H", "2H", "3H", "4H"     # 1-4 hours
```

### Daily and Above
{/* snippet: type=complete_code, needs_data=single_asset, complexity=simple */}
```python
"1D"                       # 1 day
"1W-MON", "1W-FRI"        # Week (Monday/Friday start)
"1ME", "1MS"              # Month end/start
"1QE", "1QS"              # Quarter end/start
"1YE", "1YS"              # Year end/start
```

<Callout type="warning" title="Use Exact Format">
Timeframes are case-insensitive but must use exact units:
- Valid: `"1D"`, `"1H"`, `"1W-MON"`, `"5Min"`
- Invalid: `"1Day"`, `"1Hour"`, `"1Week"`
</Callout>

## Multi-Timeframe Pattern

Higher timeframe for trend, lower timeframe for entry:

{/* snippet: type=complete_code, needs_data=single_asset, timeframe=1D, complexity=medium */}
```python
src = market_data_source(timeframe="1D")

# Daily trend filter
daily_trend = ema(period=50, timeframe="1D")(src.c)
uptrend = src.c > daily_trend

# Intraday entry signals
fast = ema(period=12)(src.c)
slow = ema(period=26)(src.c)
entry = crossover(fast, slow)

# Trade only with daily trend
buy = entry and uptrend
trade_signal_executor()(enter_long=buy)
```

## Timeframe Requirements

**Some transforms require a timeframe** (flagged with `requiresTimeFrame=true` in metadata):

{/* snippet: type=complete_code, needs_data=single_asset, timeframe=1D, needs_sessions=true, needs_intraday=true, complexity=medium */}
```python
src = market_data_source(timeframe="1D")  # ✓ Required - data source needs timeframe

# ✗ Error: Missing timeframe
gaps = session_gap(fill_percent=100)()

# ✓ Correct: Explicit timeframe
gaps = session_gap(fill_percent=100, timeframe="1Min")()

# ✓ Also correct: Inherits from src
gaps = session_gap(fill_percent=100)(src.c)
```

**Transforms requiring timeframes:**
- Data sources: `market_data_source`, `vwap`, `trade_count`
- Chart patterns: `swing_highs_lows`, `head_and_shoulders`, `triangles`, `flags`
- Calendar effects: `day_of_week`, `week_of_month`, `holiday`

<Callout type="warning" title="When to Set Timeframes">
**Always explicitly set timeframes for:**
- Data sources: `market_data_source(timeframe="1D")`
- Transforms with NO inputs: `day_of_week(timeframe="1D")`
- Transforms requiring timeframes with only constant inputs

**Timeframe inheritance only works when transforms have inputs:**
1. **Explicit timeframe wins**: `ema(period=20, timeframe="1D")(src.c)` uses `"1D"`
2. **Inherits from inputs**: `rsi(period=14)(src.c)` inherits from `src`
3. **Multiple inputs**: Takes the coarsest resolution (daily > hourly > minute)

If a transform has no inputs to inherit from, you **must** set `timeframe=` explicitly.
</Callout>
