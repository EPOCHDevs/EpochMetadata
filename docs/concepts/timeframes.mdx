# Timeframes

Multi-timeframe analysis through automatic data resampling. Execute transforms on any timeframe regardless of strategy base resolution.

---

## Overview

**Timeframe parameters enable multi-timeframe strategies** by automatically resampling data to the specified resolution before transform execution. This allows combining trend analysis on higher timeframes with entry signals on lower timeframes, or vice versa.

**Key Capability:** Any transform can operate on any timeframe. The framework handles all resampling automatically - you simply specify the desired timeframe parameter.

---

## Standard Timeframes

### Intraday
Minutes and hours for intraday trading:
- **Minutes**: 1Min, 2Min, 3Min, 5Min, 10Min, 15Min, 30Min, 45Min
- **Hours**: 1H, 2H, 3H, 4H

### Daily and Above
Daily, weekly, monthly, quarterly, and yearly resolutions:
- **Daily**: 1D
- **Weekly**: 1W-MON, 1W-FRI (Monday or Friday start)
- **Monthly**: 1ME (end), 1MS (start)
- **Quarterly**: 1QE (end), 1QS (start)
- **Yearly**: 1YE (end), 1YS (start)

<Callout type="warning" title="Use Exact Format">
Timeframes are case-insensitive but must use exact units. Valid: `"1D"`, `"1H"`, `"1W-MON"`, `"5Min"`. Invalid: `"1Day"`, `"1Hour"`, `"1Week"`.
</Callout>

---

## Advanced Timeframes

### Week Anchoring
Weekly bars can start on specific weekdays or specific occurrences within a month:
- **Day anchoring**: 1W-MON, 1W-FRI, 1W-SUN
- **Month-relative anchoring**: 1W-MON-1st (first Monday), 1W-MON-2nd (second Monday), 1W-FRI-Last (last Friday)

### Month/Quarter/Year Anchoring
Period-end vs period-start conventions:
- **Month**: 1MS (start), 1ME (end)
- **Quarter**: 1QS (start), 1QE (end)
- **Year**: 1YS (start), 1YE (end)

**Applications:** Align with fundamental data releases, earnings calendars, or specific trading schedules.

## Multi-Timeframe Strategies

**Common Pattern:** Use higher timeframe for trend filtering and lower timeframe for precise entry timing.

**Strategy Types:**
- **Trend + Entry**: Daily trend direction filters intraday signals
- **Multiple Confirmations**: Weekly regime + daily momentum + hourly execution
- **Adaptive Resolution**: Switch timeframes based on volatility or market conditions

**Applications:** Reduce false signals by confirming lower timeframe signals against higher timeframe context, improve entry timing without losing trend perspective, build hierarchical signal systems.

## Timeframe Requirements

**Transforms flagged with `requiresTimeFrame=true` must have an explicit timeframe specified** when they have no inputs or only constant inputs.

### Data Sources
All data sources require explicit timeframes:
- `market_data_source` - Primary OHLCV data
- `vwap`, `trade_count` - Supplemental bar metrics
- Other data source transforms

### Pattern Detection
Chart pattern transforms require timeframes:
- `swing_highs_lows`, `head_and_shoulders`, `triangles`, `flags`

### Calendar Effects
Time-based transforms require timeframes:
- `day_of_week`, `week_of_month`, `holiday`

### Session Transforms
Session-based transforms require timeframes:
- `session_gap`, `sessions`, `time_of_day`

---

## Timeframe Inheritance

**Transforms automatically inherit timeframes from their inputs** when:
1. Transform has at least one input
2. No explicit timeframe parameter is set
3. Input has a known timeframe

**Explicit timeframes always override inherited timeframes.** When transforms have multiple inputs with different timeframes, the coarsest resolution is used (daily > hourly > minute).

**When inheritance fails:** Transforms with no inputs or only constant inputs cannot inherit timeframes and will fail validation if `requiresTimeFrame=true`. Always set explicit timeframes for data sources and transforms with no inputs.

<Callout type="warning" title="When to Set Timeframes">
**Always explicitly set timeframes for:**
- Data sources: `market_data_source(timeframe="1D")`
- Transforms with NO inputs: `day_of_week(timeframe="1D")`

**Timeframe inheritance only works when transforms receive inputs from other transforms with known timeframes. If a transform has no inputs or cannot determine a timeframe from its inputs, you must set the timeframe explicitly.**
</Callout>
